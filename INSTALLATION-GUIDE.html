Overview

The School Management System is a complete web-based application built with Laravel (PHP) and MySQL. It provides tools for managing students, fees, attendance, homework, notices, report cards, leave applications, and more.

The system comes with:

  • Web Application — Full-featured responsive web app (looks like a native Android app on mobile)
  • Android App — WebView-based Android wrapper for native app experience
  • Web Installer — Browser-based setup wizard for easy deployment

Hostinger Installation

Follow these steps to deploy on Hostinger shared hosting:

  1. 1 Create a MySQL Database
    • Log into Hostinger hPanel
    • Go to Databases → MySQL Databases
    • Create a new database (e.g., school_management)
    • Note down: Database Name, Username, Password, and Host (usually localhost)
  2. 2 Upload Files
    • Go to Files → File Manager
    • Navigate to public_html (or your domain's root folder)
    • Upload the SchoolMS.zip file
    • Right-click the zip → Extract
    • You should now have: public_html/installer.php and public_html/school-management/
  3. 3 Set Document Root (Important!)

    Laravel's entry point is the public folder. You need to point your domain to it:

    • Method 1: .htaccess (Easiest)
    • Method 2: Change Doc Root

    Create a .htaccess file in public_html/ with this content:

    # public_html/.htaccess <IfModule mod_rewrite.c> RewriteEngine On # Redirect all requests to school-management/public/ RewriteRule ^$ school-management/public/ [L] RewriteRule ^(.*)$ school-management/public/$1 [L] </IfModule>

    In Hostinger hPanel:

    • Go to Websites → your domain → Advanced → Folder Index
    • Change the document root to public_html/school-management/public
  4. 4 Run the Web Installer
    • Open your browser and go to:
      https://yourdomain.com/installer.php
    • The installer will check requirements, ask for database credentials, and set up everything
    • After installation, delete installer.php for security
  5. 5 Set Folder Permissions

    If you get permission errors, set these folders to 775 via File Manager:

    school-management/storage/ → 775 school-management/bootstrap/cache/ → 775
Done! Your School Management System should now be live at your domain. Log in with the admin credentials you set during installation.

cPanel Installation (GoDaddy, Bluehost, etc.)

If your hosting uses cPanel instead of hPanel:

  1. 1 Create Database
    • cPanel → MySQL Databases
    • Create database, create user, add user to database with ALL PRIVILEGES
  2. 2 Upload & Extract
    • cPanel → File Manager → public_html
    • Upload SchoolMS.zip and extract it
  3. 3 Set Document Root

    Create .htaccess in public_html/ (same as Hostinger Method 1 above)

  4. 4 Run Installer

    Visit https://yourdomain.com/installer.php and follow the wizard

Manual / VPS Installation

For VPS (DigitalOcean, AWS, Linode) or local development:

Prerequisites

# Install PHP 8.2+ with required extensions sudo apt update sudo apt install php8.3 php8.3-cli php8.3-mysql php8.3-mbstring php8.3-xml php8.3-curl php8.3-zip php8.3-gd php8.3-fpm # Install MySQL sudo apt install mysql-server # Install Composer (if not installed) curl -sS https://getcomposer.org/installer | php sudo mv composer.phar /usr/local/bin/composer

Setup Steps

# 1. Extract or clone the project cd /var/www/ unzip SchoolMS.zip cd school-management # 2. Install PHP dependencies composer install --optimize-autoloader --no-dev # 3. Copy and configure environment file cp .env.example .env nano .env # Update: DB_DATABASE, DB_USERNAME, DB_PASSWORD, APP_URL # 4. Generate application key php artisan key:generate # 5. Create database mysql -u root -p -e "CREATE DATABASE school_management;" # 6. Run migrations and seed php artisan migrate --seed # 7. Set permissions sudo chown -R www-data:www-data storage bootstrap/cache sudo chmod -R 775 storage bootstrap/cache # 8. Optimize for production php artisan config:cache php artisan route:cache php artisan view:cache # 9. Set up Nginx or Apache to point to the public/ folder

Nginx Configuration

server { listen 80; server_name yourdomain.com; root /var/www/school-management/public; index index.php; location / { try_files $uri $uri/ /index.php?$query_string; } location ~ \.php$ { fastcgi_pass unix:/var/run/php/php8.3-fpm.sock; fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name; include fastcgi_params; } location ~ /\.(?!well-known) { deny all; } }

Web Installer Guide

The web installer (installer.php) automates the entire setup process through your browser.

How It Works

  1. Step 1: Checks server requirements (PHP version, extensions, folder permissions)
  2. Step 2: Asks for MySQL database credentials and tests the connection
  3. Step 3: Configures admin account and application URL
  4. Step 4: Creates .env, runs database migrations, creates admin user, optimizes caches
  5. Step 5: Installation complete!
Security Warning Delete installer.php immediately after installation! Leaving it on the server is a security risk as anyone could re-install and overwrite your database.

Troubleshooting

Common Issues

  • Check that storage/ and bootstrap/cache/ folders have 775 permissions
  • Ensure .env file exists and has correct database credentials
  • Check PHP error logs: storage/logs/laravel.log
  • Run: php artisan config:clear

  • Enable APP_DEBUG=true temporarily in .env to see errors
  • Check if APP_KEY is set in .env — if not, run php artisan key:generate
  • Make sure .htaccess is properly redirecting to school-management/public/

  • Verify database credentials in .env
  • On shared hosting, the DB host might be different from localhost — check your hosting panel
  • Make sure the database user has permissions on the database
  • After changing .env, clear cache: php artisan config:clear

  • Set APP_URL to your full https:// URL
  • If using Cloudflare or SSL proxy, add to .env:
    ASSET_URL=https://yourdomain.com
  • Force HTTPS in bootstrap/app.php if behind a proxy

  • Make sure installer.php is in the same directory as school-management/ folder
  • Access it directly: https://yourdomain.com/installer.php (not through the Laravel app)
  • If .htaccess is already redirecting, temporarily rename it and access installer directly

Android App Overview

The Android app is a WebView wrapper that loads your school management web application inside a native Android shell. This gives users a native app feel with:

  • App icon on the home screen
  • Splash screen on launch
  • Pull-to-refresh
  • Hardware back button navigation
  • Offline error screen with retry
  • No browser URL bar — feels like a native app

Android Studio Setup

Prerequisites

  • Android Studio (Arctic Fox or later)
  • Android SDK 34
  • Java 8+ (bundled with Android Studio)

Steps

  1. 1 Open Project
    • Open Android Studio
    • Select File → Open
    • Navigate to the android-app/ folder and select it
    • Wait for Gradle sync to complete (this may take a few minutes the first time)
  2. 2 Configure Server URL

    Open app/build.gradle and change the BASE_URL:

    // Change this to your actual server URL buildConfigField "String", "BASE_URL", "\"https://yourschool.com\"" // For local development with emulator: // buildConfigField "String", "BASE_URL", "\"http://10.0.2.2:8000\""
  3. 3 Run on Device/Emulator
    • Connect an Android phone via USB (enable USB Debugging in Developer Options)
    • Or create an AVD (Android Virtual Device) emulator
    • Click the Run button (green triangle) or press Shift+F10

Android App Configuration

Change App Name

Edit app/src/main/res/values/strings.xml:

<string name="app_name">Your School Name</string>

Change App Colors

Edit app/src/main/res/values/colors.xml:

<color name="primary">#3b82f6</color> <!-- Main color --> <color name="primary_dark">#2563eb</color> <!-- Status bar -->

Change App Icon

Replace the icon files in the mipmap folders, or use Android Studio's Image Asset Studio:

  • Right-click res folder → New → Image Asset
  • Select your school logo and generate all sizes automatically

Change Package Name (for Play Store)

If you want a unique package name (required for Play Store):

  1. In app/build.gradle, change applicationId:
    applicationId "com.yourschool.management"
  2. Refactor the Java package: right-click on com.schoolms.app in the project tree → Refactor → Rename

Build APK

Debug APK (for testing)

# From command line (in android-app/ folder): ./gradlew assembleDebug # APK location: app/build/outputs/apk/debug/app-debug.apk

Signed Release APK (for distribution)

  1. 1 Generate Keystore

    In Android Studio: Build → Generate Signed Bundle / APK

    • Select APK
    • Click Create new... to create a keystore
    • Fill in the details and save the keystore file securely
  2. 2 Build Signed APK
    • Select the keystore you just created
    • Choose release build type
    • Click Finish
  3. 3 Find the APK

    The signed APK will be at:
    app/build/outputs/apk/release/app-release.apk

Important Keep your keystore file safe! You need it to publish updates to the Play Store. If you lose it, you cannot update your app.

Publish to Play Store

  1. Create a Google Play Console developer account ($25 one-time fee)
  2. Create a new app and fill in the listing details (title, description, screenshots)
  3. Upload the signed APK or App Bundle
  4. Complete the content rating questionnaire
  5. Set pricing (free) and distribution countries
  6. Submit for review
Screenshots Tip Take screenshots of the app on your phone (Power + Volume Down) and upload them to the Play Store listing. You need at least 2 screenshots.

Security Notes

  • Delete installer.php after installation
  • Set APP_DEBUG=false in production
  • Set APP_ENV=production
  • Use strong admin passwords
  • Keep PHP and Laravel updated
  • Use HTTPS (most hosting provides free SSL via Let's Encrypt)
  • The .env file should never be publicly accessible (Laravel's public/ folder protects this)
  • Regularly backup your database
Backup Tip Most hosting panels have automatic backup options. Enable daily backups for your database and files.
